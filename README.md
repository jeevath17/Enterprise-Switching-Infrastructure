# Enterprise Switching Infrastructure
 Using Cisco Packet Tracer

## 📌 Project Overview

This project focuses on building an enterprise switching network using Cisco Packet Tracer. The network incorporates multiple VLANs, Inter-VLAN Routing, EtherChannel, DHCP, DHCP Relay, DNS, and HTTP (IIS) services to replicate a practical enterprise networking environment.

The primary goal is to establish secure network segmentation, ensure efficient communication between departments, provide centralized IP address allocation, improve network availability, and enable web access through domain names.

---

## 🏢 Network Architecture

The enterprise network is designed with the following components:

* Access Layer Switches
* Multiple Client PCs
* Distribution Layer
* DHCP Server
* Core Layer Switch
* Multiple VLANs
* HTTP (IIS) Server
* Data Center
* DNS Server

---

* EtherChannel using PAgP

## ⚙️ Configured Services

### Inter-VLAN Routing

* Implemented on the Layer 3 Switch to allow communication between different VLANs.

### DHCP

* DNS server information distributed to clients
* Automatic IP address assignment
* Default gateway configuration

### DHCP Relay

* IP Helper Address configured to forward DHCP requests from remote VLANs to the centralized DHCP server.

### DNS Server

* Custom DNS records created for enterprise websites
* Resolves domain names into IP addresses

### HTTP (IIS) Server

Custom webpages were hosted, including:

* mcsa.com
* ccna.com

### EtherChannel

Implemented using:

* PAgP
* LACP

to improve bandwidth utilization and provide redundancy between switches.
